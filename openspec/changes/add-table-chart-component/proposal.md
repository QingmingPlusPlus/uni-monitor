## Why

当前工厂大屏需要一种统一的数据分析卡片，用同一块区域同时展示可配置的指标表格和趋势图，方便后续页面复用入库计划、实绩、差异、达成率等类似业务视图。

该组件需要先抽象为独立能力，避免每个页面重复拼表格、图表和标题区，并为后续接入不同业务数据留出稳定 props 契约。

## What Changes

- 新增一个表格加图表的复合展示组件，视觉参考用户提供的分析卡片示意图。
- 组件顶部展示标题和可选副标题，右上角预留展开按钮事件处理函数，当前实现为空函数占位。
- 组件上半部分展示表格，表格行配置、列配置和表格数据均由 props 传入。
- 组件下半部分展示 ECharts 图表，基础 `chartOptions` 和独立 `chartData` 均由 props 传入。
- 组件需要遵循现有 `DESIGN.md` 的工业大屏设计系统，使用已有颜色、间距、字号和低动效策略。

## Capabilities

### New Capabilities

- `configurable-table-chart-card`: 定义一个可复用的表格加 ECharts 图表卡片能力，包括标题区、可配置表格、可配置图表、空状态和展开占位交互。

### Modified Capabilities

- 无。

## Impact

- 影响前端组件目录，预计新增 Vue 3/uni-app 单文件组件和必要类型定义。
- 影响依赖管理，当前 `package.json` 尚未包含 ECharts 相关依赖，实现阶段需要确认并添加适合 uni-app 的 ECharts 集成方案。
- 影响使用方页面的数据契约，调用方需要按组件 props 提供标题、表格配置、表格数据、图表配置和图表数据。
- 不改变现有页面路由、导航逻辑和三维度页面的信息架构。
