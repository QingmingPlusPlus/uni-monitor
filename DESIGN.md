# Uni Monitor 设计系统

## 1. 氛围与识别

Uni Monitor 是面向工厂现场的 2K 触摸大屏可视化界面：浅色工业底盘、白色数据面板、蓝色操作反馈，以及绿/黄/红状态编码。界面优先服务远距离扫视和现场触控，不照搬参考图的小字号和密集控件，而是在相同信息结构上提高字号、按钮命中面积和面板节奏。

当前核心信息架构围绕 **部门、工序、设备** 三个维度展开：部门页左侧为 `css-map` 厂区地图，右侧为 `2列 x 2排` 的人员出勤、出勤率、人员明细和入库计划面板；工序页左侧为 `css-map` 厂区地图，右侧直接进入多列瀑布流，不在瀑布流上方展示 KPI 网格；设备页为单设备详情分析面板。

### 规则

- 页面基底保持浅灰，数据承载面使用白色，避免大屏长时间观看疲劳。
- 蓝色只表达操作、选中、路由和按钮状态，不参与设备健康判断。
- 设备健康只使用成功、警告、危险和中性状态色，颜色必须配文字；`css-map` 的切替和清扫属于工况状态，使用专用工况色避免与健康告警混淆。
- 图表颜色从操作蓝、成功绿、危险红、警告黄中取值，不新增装饰性色。

## 3. 字体排版

### 字号层级

| 层级 | 字号 | 字重 | 行高 | 字距 | 用途 |
| --- | --- | --- | --- | --- | --- |
| 大屏页面标题 | `34px` | 800 | 1.2 | 0 | 部门/工序/设备标题 |
| 大屏面板标题 | `22px` | 800 | 1.25 | 0 | 仪表盘、详情卡标题 |
| 大屏关键数字 | `28px` | 800 | 1.1 | 0 | KPI 数值 |
| 大屏卡片标题 | `30px` | 700 | 1.25 | 0 | `TableChartCard` 标题 |
| 大屏正文 | `17px` | 500 | 1.45 | 0 | 描述、说明 |
| 大屏标签 | `15px` | 600 | 1.35 | 0 | 表格、图例、按钮 |
| 移动 H1 | `46rpx` | 800 | 1.2 | 0 | 窄屏头部标题 |
| 移动正文 | `26rpx` | 500 | 1.55 | 0 | 窄屏说明 |

### 字体栈

- 主字体：系统 UI 字体。
- 数字：系统字体并启用 `font-variant-numeric: tabular-nums`。

### 规则

- 2K 大屏正文不低于 `15px`，核心指标不低于 `24px`。
- 地图按钮和下拉控件不低于 `15px`，确保触屏点击和远距离识别。
- 不使用负字距；所有标题、标签和按钮字距为 `0`。

## 4. 间距与布局

### 基础单位

移动端使用 `8rpx` 基础单位；H5 宽屏使用 `8px` 节奏。

| 设计变量 | 移动值 | 宽屏值 | 用途 |
| --- | --- | --- | --- |
| `--space-1` | `8rpx` | `8px` | 紧凑行内间距 |
| `--space-2` | `16rpx` | `12px` | 行间分隔 |
| `--space-3` | `24rpx` | `16px` | 卡片内边距 |
| `--space-4` | `32rpx` | `20px` | 面板间距 |
| `--space-5` | `40rpx` | `24px` | 页面边距 |
| `--space-6` | `48rpx` | `32px` | 主要分区 |

### 栅格

- 部门/工序大屏布局：顶部信息栏 + 左侧 `css-map` + 右侧可滚动仪表盘。
- `1024px` 以上进入左右两栏；部门页地图占左侧三分之一，右侧瀑布流占三分之二；工序页在 `1920px` 以上优化为地图和仪表盘接近等宽。
- 设备页为全宽详情网格，顶部 KPI 后接计划、原因、时间轴和周期面板。
- 移动端退化为单列纵向信息流。

### 规则

- `css-map` 是部门/工序页的唯一主展示区，不再使用占位方块。
- 右侧仪表盘滚动承载部门业务组件、工序 `TableChartCard`；部门页宽屏为 `2列 x 2排`，窄屏退化为单列。
- 部门页人员出勤率推移表与入库计划推移表只展示周数据；周数据由月内每日数据源按工序级周配置汇总生成。
- 固定格式控件使用稳定尺寸，避免按钮、图例、KPI 动态内容造成布局跳动。

## 5. 组件

### `css-map`

- **结构**：地图场景、部门/工序下拉、图例、屏幕操作按钮、设备节点。
- **交互**：下拉触发真实路由；地图区域只做视觉聚焦；设备双击进入设备页。
- **屏幕按钮**：固定步长平移、放大、缩小、重置、适配、点选模式。
- **状态**：点选模式必须有可见蓝色反馈。

### `TableChartCard`

- **结构**：标题区、上方表格、下方图表、展开弹层。
- **使用**：所有“上方 table / 下方 chart”的面板都通过配置传入，不复制结构。
- **尺寸**：大屏单卡保持约 `480-520px` 高，支持右侧滚动承载多卡。

### KPI 网格

- **结构**：标签、关键数字、说明。
- **状态**：左边框使用操作/成功/警告/危险/中性色区分语义。
- **字体**：数字使用大屏关键数字层级。

### 设备详情面板

- **结构**：返回按钮、设备标题、KPI、计划表、原因表、时间轴、周期表。
- **导航**：返回按钮根据 `from` 和设备所属工序回到部门或工序维度。
- **数据**：当前为降级演示数据，页面不显示演示标识；后续通过 URL 中的 `deviceId` 查询真实接口。

## 6. 动效与交互

| 类型 | 时长 | 缓动 | 用途 |
| --- | --- | --- | --- |
| 微交互 | `120ms` | ease-out | 按钮按压、点选反馈 |
| 标准 | `200ms` | ease-in-out | 面板展开、状态切换 |

### 规则

- 触屏按钮只使用固定步长，避免长按连续移动造成误操作。
- 所有触摸反馈只使用 `transform` 与 `opacity`。
- 设备双击和点选模式都能进入设备详情，双击作为快捷入口保留。

## 7. 景深与表面

### 策略

以边框和轻量阴影区分层级，地图节点允许轻阴影帮助和底图分离。

| 类型 | 值 | 用途 |
| --- | --- | --- |
| 默认 | `1px solid var(--um-color-border)` | 面板、指标瓦片 |
| 操作 | `1px solid var(--um-color-operation)` | 选中按钮、返回按钮 |
| 分隔 | `1px solid var(--um-color-rail)` | 表格和轨道 |
| 轻阴影 | `0 8px 20px rgba(21, 43, 70, 0.12)` | 浮动地图控件 |
