# 工厂可视化三维度说明

本项目是一个工厂可视化看板，用于在大屏上集中呈现工厂项目的运行状态。整体信息架构围绕三个维度展开：**部门**、**工序**、**设备**。

## 三个维度

### 1. 部门维度（Department）

- 路由：`pages/department/index`
- 描述：以部门为粒度展示工厂的组织架构与整体状态。每个部门负责一个或多个工序，部门维度关注的是"谁在负责什么"。
- 页面布局：左侧为主展示区（`StagePlaceholder`），右侧为部门卡片瀑布流。

### 2. 工序维度（Process）

- 路由：`pages/process/index`
- 描述：以工序为粒度展示生产流程。每个工序包含若干设备，工序维度关注的是"怎么做、做到什么程度"。
- 页面布局：与部门维度相同的双列布局，左侧主展示区，右侧工序卡片瀑布流。

### 3. 设备维度（Equipment）

- 路由：`pages/equipment/index`
- 描述：以设备为粒度展示具体的机器、产线单元。设备维度关注的是"具体哪台设备在做什么"。
- 页面布局：独立的全宽页面，无左侧主展示区，直接展示设备卡片瀑布流，并提供返回按钮。

## "工序"与"设备"的别名关系

在某些业务场景或历史语境中，**"工序"也被称为"设备"**，或者两者被混用。这是因为：

- 在较粗的粒度下，一个工序可能只对应一台关键设备（例如"冲压工序"≈"冲压机"）。
- 在部分老系统或口头表达中，工人习惯用设备名代指工序。

在本项目中，**工序和设备属于同一层级的不同视角**：
- 工序视角强调流程、节拍、质量流转。
- 设备视角强调具体机器的运行状态、参数与维护。

两者在层级上处于同一位置，但在展示粒度与关注点上有区别。

## 层级关系

```
部门（Department）
  └── 工序（Process）—— 一个部门负责一个或多个工序
        └── 设备（Equipment）—— 一个工序包含若干设备
```

- **部门 → 工序**：部门是组织单元，工序是生产流程单元。
- **工序 → 设备**：工序由若干设备协同完成。

## 导航逻辑

- 在部门或工序页面的左侧主展示区（`StagePlaceholder`）**双击**，可进入设备维度页面。
- 设备维度页面提供"返回上一维度"按钮，通过 `uni.navigateBack()` 返回之前的页面。
