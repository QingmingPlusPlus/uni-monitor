# 工厂可视化三维度说明

本项目是一个工厂可视化看板，用于在 2K 触摸大屏上集中呈现工厂运行状态。整体信息架构围绕三个维度展开：**部门**、**工序**、**设备**。

## 三个维度

### 1. 部门维度（Department）

- 路由：`pages/department/index?departmentId=department1`
- 描述：以部门为粒度展示当前科室覆盖工序的地图、稼动、人员出勤、趋势和计划实绩。
- 页面布局：顶部为 `Omni Monitor` + 当前部门告警跑马灯；主体为左右两栏，左侧约三分之一展示 `css-map`，右侧约三分之二为纵向组件栈。
- 右侧组件顺序：信息汇总、生产线稼动情况、人员出勤情况、人员明细及状态、出勤率推移表、入库计划实绩推移表。
- 数据范围：生产线稼动、人员出勤和人员明细仅展示当前部门相关数据；其中生产线稼动按当前部门下每个工序展示一行。
- 特殊规则：`departmentId === 'department1'`（制造1课）时不展示入库计划实绩推移表。
- 状态来源：`departmentId` query 是页面状态源。
- 长屏刷新：部门页按设备本地时间每日 06:20 软刷新一次显示数据；手动刷新按卡片粒度重新拉取被点击卡片的数据，不触发整页重载，不影响左侧 `css-map` 的部门/工序选择。

### 2. 工序维度（Process）

- 路由：`pages/process/index?processId=pretreatment1`
- 描述：以工序为粒度展示当前工序的地图、稼动、人员出勤、趋势和生产计划实绩。
- 页面布局：与部门维度一致，左侧约三分之一展示 `css-map`，右侧约三分之二为纵向组件栈。
- 右侧组件顺序：信息汇总、生产线稼动情况、人员出勤情况、人员明细及状态、出勤率推移表、入库计划实绩推移表、生产计划实绩推移表。
- 数据范围：生产线稼动、人员出勤和人员明细仅展示当前选择工序；生产线稼动只展示当前工序一行；入库计划实绩按当前工序所属部门聚合。
- 状态来源：`processId` query 是页面状态源。

### 3. 设备维度（Equipment）

- 路由：`pages/equipment/index?deviceId=1101&from=department`
- 描述：以单个设备为粒度展示停机、计划、损耗、原因、时间轴和周期信息。
- 页面布局：顶部为返回按钮 + `Omni Monitor` + 当前设备告警跑马灯；主体为全宽详情分析面板。
- 状态来源：`deviceId` query 是设备主键；`from` 用于返回部门或工序维度。

## 地图交互

- 地图数据源：`public/factory-map/devices.json`；H5 构建同时保留 `src/static/factory-map/devices.json` 作为 Uni-app 静态资源副本。
- 选择器配置源：`public/factory-map/selection.json`；H5 构建同时保留 `src/static/factory-map/selection.json`。
- 地图默认嵌入在部门/工序维度左侧，支持部门、工序选择和设备双击进入详情。
- 左侧地图提供展开按钮，点击后打开 80vw × 80vh 弹窗，弹窗内展示放大的 `css-map`。
- 地图弹窗内的部门选择、工序选择、清空工序和设备打开事件均不驱动页面路由切换，只用于弹窗内临时查看。
- 地图实时数据由 `src/components/css-map/css3dMapLiveData.ts` 聚合：设备工作状态、负荷率、人员配置和 5M 变化点均优先使用真实接口。

## 推移表展示规则

- 推移类数据后端按日返回，前端按当前查询到的月周配置汇总成月、周和日。
- 右侧小表列只展示：月列、接口周配置对应的周列、当前周工作日日列；即使日列对应日期尚未到达，也要展示列并以空值呈现；周六、周日因不出勤不展示；右侧小表不横向滚动。
- 展开弹窗展示完整表格和完整图表：表格列为月、全部周、当前月全部日；列宽不足时允许横向滚动。
- 未到日期或接口返回 0 且不应视为有效数据时，chart 数据使用 `null` 跳过该点，避免折线骤降到 0。
- 出勤率推移中的利计出勤率固定为 91.0%；表格只在月列显示一个 `91.0%`，图表中展示一条 91.0% 红色目标线，并在线末端标注 `91%`。
- 出勤率推移表的月、周人数聚合显示为整数；其中直接出勤人数月/周平均剔除直接出勤为 0 的天，直接实际出勤率月/周聚合也剔除这些天；百分比统一显示一位小数。
- 入库计划实绩推移表和生产计划实绩推移表的表格保留月列，折线图不展示月列。

## 层级关系

```text
部门（Department）
  └── 工序（Process）—— 一个部门负责一个或多个工序
        └── 设备（Equipment）—— 一个工序包含若干设备
```

- **部门 → 工序**：由 `public/factory-map/selection.json` 维护部门与工序的聚合关系。
- **工序 → 设备**：由 `devices.json` 中设备的 `section` 字段关联。
- **设备详情**：以 `devices.json` 中的设备 `id` 作为 URL 主状态。

## 页面路由一览

| 路由 | 维度 | 状态 query | 布局 |
| --- | --- | --- | --- |
| `pages/department/index` | 部门 | `departmentId` | 告警栏 + 左侧 `css-map` + 右侧纵向组件栈 |
| `pages/process/index` | 工序 | `processId` | 告警栏 + 左侧 `css-map` + 右侧纵向组件栈 |
| `pages/equipment/index` | 设备 | `deviceId`、`from` | 带返回按钮的告警栏 + 单设备详情分析面板 |
